# Git环境变量配置指南

## 一、确认Git安装情况

根据检查，您已经成功安装了Git for Windows，安装目录为：

```
E:\abc\Git
```

Git的可执行文件位于：

```
E:\abc\Git\cmd\git.exe
```

## 二、为什么需要配置环境变量？

目前您无法在PowerShell中直接使用`git`命令，是因为Windows系统不知道`git`命令对应的可执行文件位置。配置环境变量可以让系统在任何目录下都能找到并执行Git命令。

## 三、配置Git环境变量步骤

### 步骤1：打开环境变量设置

1. 按下 `Win + R` 键，打开「运行」对话框
2. 输入 `sysdm.cpl` 并按下 Enter 键
3. 在「系统属性」窗口中，点击「高级」选项卡
4. 点击「环境变量」按钮

### 步骤2：配置Path环境变量

1. 在「环境变量」窗口中，找到「系统变量」区域
2. 找到并双击「Path」变量
3. 在「编辑环境变量」窗口中，点击「新建」按钮
4. 输入Git的cmd目录路径：
   ```
   E:\abc\Git\cmd
   ```
5. 点击「确定」保存设置

### 步骤3：验证环境变量配置

1. 打开一个**新的**PowerShell窗口（重要：必须是新窗口，旧窗口不会加载新的环境变量）
2. 输入以下命令验证Git是否可以正常使用：
   ```
   git --version
   ```
3. 如果显示Git版本信息，说明配置成功！

## 四、使用Git命令行

配置成功后，您可以在任何目录下使用Git命令，例如：

- 查看Git版本：`git --version`
- 初始化Git仓库：`git init`
- 查看当前状态：`git status`

## 五、下一步操作

完成环境变量配置后，您可以继续使用命令行Git发布项目，或者使用之前提供的GitHub Desktop图形化工具。

如果您想使用命令行发布项目，可以参考之前创建的「GITHUB发布指南.md」文件。

## 六、常见问题

### 1. 配置后仍然无法使用git命令

- 确保您打开了新的PowerShell窗口
- 检查环境变量路径是否正确：`E:\abc\Git\cmd`
- 重启计算机后再次尝试

### 2. 如何查看当前环境变量？

在PowerShell中输入以下命令：

```
$env:Path -split ';' | Select-String -Pattern "git"
```

### 3. 如何卸载Git？

如果需要卸载Git，可以运行以下程序：

```
E:\abc\Git\unins000.exe
```
